## Phases

- [ ] **Phase 7: Detailed Feedback (M5, N4, N1)** — Breakdown criteria UI, technology badges, and Markdown export.
- [ ] **Phase 8: Active Control (M8, N2)** — Per-question timer logic and Simulation Mode visibility.

- [ ] **Phase 10: Advanced Analytics (M6, N3)** — Criteria aggregation in dashboard and streak logic fix.

## Phase Details

### Phase 7: Detailed Feedback
**Goal:** Show users *why* they got a score.
**Requirements:** UI-01, UI-02, UI-08, UI-06, LOGIC-02.
**Success Criteria:** Results page shows 4-bar breakdown per question; Session cards show tech names; .md file downloads correctly.

### Phase 8: Active Control
**Goal:** Add pressure and focus to the interview simulation.
**Requirements:** UI-04, UI-05, CONFIG-01, LOGIC-01, UI-07.
**Success Criteria:** Countdown appears and resets on new questions; Timer can be disabled in config; "Simulation Mode" is easily selectable.

### Phase 9: Data & Search
**Goal:** Clean up the history and make it searchable.
**Requirements:** API-02, UI-03, UI-09, UI-10.
**Success Criteria:** Sessions can be deleted; Search shows "No results found" instead of an empty list.

### Phase 10: Advanced Analytics
**Goal:** Provide actionable weekly insights.
**Requirements:** API-01, DASH-01, LOGIC-03.
**Success Criteria:** Dashboard shows "Weakest Criteria" (e.g., Depth); Streak counter correctly identifies consecutive study days.
